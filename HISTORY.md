# History

1.0.3
==================

1. Transactional outbox: Возможность создания пакета событий в одной транзакции.
2. Transactional outbox: Возможность хранения метаданных у события.
3. Transactional outbox: Возможность указания стратегии обработки при создании событий. У каждого события может быть либо
   своя стратегии отправки, либо стратегия по умолчанию.
4. Transactional outbox: Исправлена транзакционность контекстного менеджера outbox_utils.OutboxEvent. 

1.0.2
==================

1. Обеспечивает работу базового алгоритма
   паттерна [Transactional Outbox](https://microservices.io/patterns/data/transactional-outbox.html)
2. Предоставляет API регистрации событий с дальнейшей их отправкой во внешнюю систему
3. Предоставляет API для реализации своей собственной стратегии отправки.
4. Возможность повторной отправки.
5. Предоставляет утилитарные задачи, поддерживающие актуальность событий и своевременную их удаление.

1.0.1
==================

1. Ведение истории запусков задач Celery. Отслеживание их статусов, продолжительности и параметров запуска.
2. Хранение истории за последние сутки и поиск по ней. Возможность настроить время хранения истории.
3. Работа с состоянием и результатом выполнения задач. Состояние выполнения позволит восстановить работу задачи с того,
   места, на котором она прервалась. Функции monitoring_utils.save_state и monitoring_utils.load_state
4. Возможность перезапуска задачи. Функция monitoring_utils.restart_task.
5. Возможность гарантированного прерывания длительных задач. Функции monitoring_utils.find_task, instance.revoke_task и
   Terminator.
6. Отслеживание количества попыток перезапуска задач.
7. Ведение общего репозитория с "белым" и "черным" списком видимости задач для системы мониторинга.
8. Возможность профилирования задач(в процессе реализации) в части CPU и ОЗУ.
9. Возможность динамической настройки задач в процессе выполнения из репозитория задач
